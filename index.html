talent-AIcquisition

Prototype for a rich, grounded AI talent acquisition app — experiment log and artifacts.

What this prototype is

This repo is an experiment in generating candidate assessments that are grounded in source documents (resumes, long-form writing, notes, etc.) and reviewable via citations in the UI.

Key finding: grounding metadata can be sparse

In practice, the grounding metadata returned by Google File Search is often sparse relative to the number of factual claims made in an assessment. This creates a gap between “nice narrative output” and “auditable, hire-safe output.”

What we’re iterating on: tuning RAG chunking configuration and the system prompt to increase citation density/precision (work in progress).

Next iteration: two-pass generation

The next step is a two-pass approach:

  1. Pass 1 — evidence-first: retrieve and structure candidate evidence into smaller, attributable “fact units.”
  2. Pass 2 — publish-ready assessment: generate the final narrative while preserving the mapping from claims → evidence.

The intent is to improve reliability (reduce inference stretch) and make citations consistently useful for reviewers.
